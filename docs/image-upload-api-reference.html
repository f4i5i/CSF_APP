1. Overview — How Uploads Work

Critical: All Uploads Use multipart/form-data with Raw Binary

The CSF backend accepts image uploads only as raw binary files via multipart/form-data encoding.

Do NOT send images as Base64-encoded strings. The server expects the actual file bytes in a multipart form field named file.

✅ Correct — Raw Binary via multipart/form-data

// The file bytes are sent directly in the form body
POST /api/v1/photos/upload
Content-Type: multipart/form-data; boundary=----abc123

------abc123
Content-Disposition: form-data; name="file"; filename="photo.jpg"
Content-Type: image/jpeg

[RAW JPEG BINARY BYTES]
------abc123
Content-Disposition: form-data; name="class_id"

cls_abc123
------abc123--

❌ Wrong — Base64 in JSON Body

// This will be REJECTED by the server
POST /api/v1/photos/upload
Content-Type: application/json

{
  "file": "data:image/jpeg;base64,/9j/4AAQ...",
  "class_id": "cls_abc123"
}

// Server expects UploadFile, not JSON
// Response: 422 Unprocessable Entity

How the Server Processes Your Upload

Mobile sends multipart/form-data with raw file
→ FastAPI parses into UploadFile object
→ Server reads bytes: await file.read()
→ Validates content type & file size
→ (For photos) Creates 300×300 thumbnail via Pillow
→ Stores raw bytes directly in PostgreSQL BYTEA column
→ Returns JSON response with image URL paths

Key point: Images are stored as binary blobs in the database — there is no S3, CDN, or filesystem storage. Retrieval endpoints serve the bytes back directly.

Obtain the token via POST /api/v1/auth/login. Token is returned in the access_token field of the login response.

⚠️ Do NOT Manually Set Content-Type Header

When sending multipart/form-data, you must let the HTTP client set the Content-Type header automatically. The header must include a boundary parameter that matches the actual form data boundaries.

// ❌ WRONG — will cause 422 error
headers: {
  'Content-Type': 'multipart/form-data',  // Missing boundary!
  'Authorization': 'Bearer ...'
}

// ✅ CORRECT — only set Authorization, let client handle Content-Type
headers: {
  'Authorization': 'Bearer ...'
}

3. API 1: Child Profile Image Upload

POST multipart/form-data /api/v1/children/{child_id}/profile-image

Upload or replace a child's profile picture. The previous image is overwritten — no versioning.

Who Can Use This

Parent Own children only  |  Admin Owner Any child

URL Parameters

ParameterTypeRequiredDescription
child_idstring (UUID)YesThe child's unique ID

Form Fields

Field NameTypeRequiredDescription
file Binary File Yes The image file. Must be image/jpeg, image/png, or image/jpg. Max 10 MB.

What the Server Does

  1. Validates the user has access to this child (parent must own the child)
  2. Validates file content type is in allowed list
  3. Reads the binary content and checks size ≤ 10 MB
  4. Stores raw bytes in child.profile_image_data column (PostgreSQL BYTEA)
  5. Sets child.profile_image_content_type to the uploaded MIME type
  6. Sets child.profile_image to "db://{child_id}" (internal marker)
  7. Returns the full ChildResponse with profile_image set to the retrieval URL

Success Response — 200 OK

{
  "id": "a1b2c3d4-...",
  "user_id": "parent-uuid",
  "first_name": "Emma",
  "last_name": "Smith",
  "full_name": "Emma Smith",
  "date_of_birth": "2018-05-15",
  "age": 7,
  "grade": "2",
  "profile_image": "/api/v1/children/a1b2c3d4-.../profile-image",
  // ... all other child fields
}

The profile_image field in the response is the relative URL to retrieve the image. Prepend your base URL to use it.

4. API 2: Gallery Photo Upload

POST multipart/form-data /api/v1/photos/upload

Upload a photo to one or more class galleries. The server auto-generates a 300×300 thumbnail.

Who Can Use This

Coach Admin Owner   Parents cannot upload gallery photos.

Form Fields

Field NameTypeRequiredDescription
file Binary File Yes The image file. Must be image/jpeg, image/png, or image/jpg. Max 10 MB.
class_id string Conditional* Single class UUID to associate the photo with.
class_ids string (JSON array) Conditional* JSON-encoded array of class UUIDs for multi-class upload. Example: '["id1","id2"]'
category_id string No Photo category UUID (created via POST /photos/categories)

* At least one of class_id or class_ids must be provided. You can provide both — they are merged.

⚠️ Important: class_ids is a JSON String, Not an Array

Because the request is multipart/form-data (not JSON), the class_ids field must be a JSON-encoded string sent as a plain text form field.

// ✅ Correct — JSON string in a form field
formData.append('class_ids', JSON.stringify(["cls_1", "cls_2"]));

// ❌ Wrong — cannot send arrays in multipart/form-data
formData.append('class_ids', ["cls_1", "cls_2"]);

What the Server Does

  1. Validates user role is Coach, Admin, or Owner
  2. Parses class_ids JSON string (or uses class_id). Merges both if provided
  3. Validates file content type and size
  4. Opens image with Pillow (PIL) and extracts width/height dimensions
  5. Converts RGBA → RGB if needed (handles transparent PNGs)
  6. Creates a 300×300 max thumbnail (maintains aspect ratio, JPEG format, 85% quality)
  7. Stores both full image and thumbnail as binary in database
  8. Creates PhotoTarget records for each class (multi-class support)
  9. Returns PhotoResponse with computed URL fields

Success Response — 201 Created

{
  "id": "photo-uuid-123",
  "class_id": null,                          // null for multi-class photos
  "category_id": "cat-uuid-456",              // or null
  "uploaded_by": "user-uuid-789",
  "file_name": "IMG_2024.jpg",
  "file_path": null,                          // always null (DB storage)
  "thumbnail_path": null,                     // always null (DB storage)
  "file_size": 2458624,                        // bytes
  "width": 3024,                               // original pixel width
  "height": 4032,                              // original pixel height
  "content_type": "image/jpeg",
  "created_at": "2026-03-16T10:30:00Z",
  "targets": [                                 // which classes this photo belongs to
    { "id": "target-1", "photo_id": "photo-uuid-123", "class_id": "cls_1" },
    { "id": "target-2", "photo_id": "photo-uuid-123", "class_id": "cls_2" }
  ],
  "mime_type": "image/jpeg",                  // computed, same as content_type
  "image_url": "/api/v1/photos/photo-uuid-123/image",         // full-size URL
  "thumbnail_url": "/api/v1/photos/photo-uuid-123/thumbnail"   // 300×300 thumbnail URL
}

Note: image_base64 and thumbnail_base64 fields exist in the schema but are excluded from the JSON response for performance (the binary data fields are marked exclude=True). Always use the URL endpoints to fetch images.

5. API 3: Announcement Attachment Upload

POST multipart/form-data /api/v1/announcements/{announcement_id}/attachments

Upload a file attachment (image or PDF) to an existing announcement.

Who Can Use This

Coach Author of the announcement only  |  Admin Owner Any announcement

URL Parameters

ParameterTypeRequiredDescription
announcement_idstring (UUID)YesThe announcement to attach the file to

Form Fields

Field NameTypeRequiredAccepted Content Types
file Binary File Yes image/jpeg, image/png, image/jpg, application/pdf

What the Server Does

  1. Validates the announcement exists and is not deleted
  2. Checks user is the author OR an admin/owner
  3. Validates file content type (images + PDF allowed)
  4. Reads binary content and checks size ≤ 10 MB
  5. Determines attachment type: IMAGE or PDF
  6. Stores raw bytes in AnnouncementAttachment.file_data (BYTEA)
  7. Returns attachment metadata with computed file_url

Success Response — 201 Created

{

  "file_size": 524288,                         // bytes

  "mime_type": "application/pdf",

  "file_url": "/api/v1/announcements/attachments/attach-uuid-456/file"

7. React Native — Complete Examples

7.1 Image Picker + Upload Helper

import * as ImagePicker from 'expo-image-picker';
import { Platform } from 'react-native';

/**
 * Pick an image from camera or gallery.
 * Returns the local URI, MIME type, and filename.
 */
async function pickImage(fromCamera = false) {
  const options = {
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    allowsEditing: true,
    quality: 0.8,          // compress to ~80% quality to stay under 10MB
  };

  const result = fromCamera
    ? await ImagePicker.launchCameraAsync(options)
    : await ImagePicker.launchImageLibraryAsync(options);

  if (result.canceled) return null;

  const asset = result.assets[0];
  return {
    uri: asset.uri,
    type: asset.mimeType || 'image/jpeg',
    name: asset.fileName || `photo_${Date.now()}.jpg`,
    fileSize: asset.fileSize,
  };
}

/**
 * Upload a file using multipart/form-data.
 *
 * @param url      Full endpoint URL
 * @param token    JWT Bearer token
 * @param fileObj  { uri, type, name } from image picker
 * @param extras   Additional form fields: { class_id: "...", category_id: "..." }
 * @returns        Parsed JSON response
 */
async function uploadFile(url, token, fileObj, extras = {}) {
  const formData = new FormData();

  // ─── THE FILE ───────────────────────────────────────────
  // React Native's FormData expects an object with { uri, type, name }
  // This is NOT a Web File/Blob — it's React Native-specific.
  // The RN networking layer reads the file from the URI and streams
  // the raw bytes into the multipart body automatically.
  formData.append('file', {
    uri: Platform.OS === 'ios'
      ? fileObj.uri.replace('file://', '')  // iOS sometimes needs file:// stripped
      : fileObj.uri,
    type: fileObj.type,     // e.g. 'image/jpeg' — server validates this
    name: fileObj.name,     // e.g. 'photo.jpg' — server stores this
  });

  // ─── ADDITIONAL FORM FIELDS ─────────────────────────────
  Object.entries(extras).forEach(([key, value]) => {
    if (value !== undefined && value !== null) {
      formData.append(key, typeof value === 'object'
        ? JSON.stringify(value)  // for arrays like class_ids
        : String(value)
      );
    }
  });

  const response = await fetch(url, {
    method: 'POST',
    headers: {
      // ⚠️ ONLY set Authorization. Do NOT set Content-Type.
      // fetch() automatically sets:
      //   Content-Type: multipart/form-data; boundary=----RNFormBoundary...
      'Authorization': `Bearer ${token}`,
    },
    body: formData,
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.detail || error.message || `Upload failed: ${response.status}`);
  }

  return await response.json();
}

8. Flutter / Dart Examples

8.1 Upload Helper Using http Package

import 'package:http/http.dart' as http;
import 'package:http_parser/http_parser.dart';
import 'dart:convert';

/// Upload a file using multipart/form-data.
///
/// [filePath] - Local file path (from image_picker or file_picker)
/// [mimeType] - e.g. 'image/jpeg', 'application/pdf'
/// [fileName] - Display name for the file
/// [extras]   - Additional form fields
Future<Map<String, dynamic>> uploadFile({
  required String url,
  required String token,
  required String filePath,
  required String mimeType,
  required String fileName,
  Map<String, String> extras = const {},
}) async {
  final request = http.MultipartRequest('POST', Uri.parse(url));

  // ─── Auth header ───────────────────────────────────────
  request.headers['Authorization'] = 'Bearer $token';
  // Do NOT set Content-Type — MultipartRequest handles it

  // ─── The file (raw binary, read from disk) ─────────────
  request.files.add(
    await http.MultipartFile.fromPath(
      'file',              // field name — must be exactly 'file'
      filePath,
      filename: fileName,
      contentType: MediaType.parse(mimeType),
    ),
  );

  // ─── Additional text fields ────────────────────────────
  extras.forEach((key, value) {
    request.fields[key] = value;
  });

  final streamedResponse = await request.send();
  final response = await http.Response.fromStream(streamedResponse);

  if (response.statusCode != 200 && response.statusCode != 201) {
    throw Exception('Upload failed (${response.statusCode}): ${response.body}');
  }

  return jsonDecode(response.body);
}

9. Swift (iOS) Examples

9.1 Upload Helper Using URLSession

import Foundation

struct UploadResult: Codable {
    let id: String
    // Add other fields as needed
}

func uploadImage(
    url: URL,
    token: String,
    imageData: Data,
    mimeType: String,          // "image/jpeg" or "image/png"
    fileName: String,
    extraFields: [String: String] = [:]
) async throws -> Data {
    let boundary = "Boundary-\(UUID().uuidString)"

    var request = URLRequest(url: url)
    request.httpMethod = "POST"
    request.setValue("Bearer \(token)", forHTTPHeaderField: "Authorization")
    // Set Content-Type WITH boundary (required for multipart)
    request.setValue(
        "multipart/form-data; boundary=\(boundary)",
        forHTTPHeaderField: "Content-Type"
    )

    var body = Data()

    // ─── The file field ─────────────────────────────────
    body.append("--\(boundary)\r\n".data(using: .utf8)!)
    body.append(
        "Content-Disposition: form-data; name=\"file\"; filename=\"\(fileName)\"\r\n"
            .data(using: .utf8)!
    )
    body.append("Content-Type: \(mimeType)\r\n\r\n".data(using: .utf8)!)
    body.append(imageData)     // raw bytes — NOT base64
    body.append("\r\n".data(using: .utf8)!)

    // ─── Extra text fields ─────────────────────────────
    for (key, value) in extraFields {
        body.append("--\(boundary)\r\n".data(using: .utf8)!)
        body.append(
            "Content-Disposition: form-data; name=\"\(key)\"\r\n\r\n"
                .data(using: .utf8)!
        )
        body.append("\(value)\r\n".data(using: .utf8)!)
    }

    // ─── Close boundary ────────────────────────────────
    body.append("--\(boundary)--\r\n".data(using: .utf8)!)

    request.httpBody = body

    let (data, response) = try await URLSession.shared.data(for: request)
    guard let httpResponse = response as? HTTPURLResponse,
          (200...201).contains(httpResponse.statusCode) else {
        throw URLError(.badServerResponse)
    }
    return data
}

10. Kotlin (Android) Examples

10.1 Upload Helper Using OkHttp

import okhttp3.*
import okhttp3.MediaType.Companion.toMediaType
import okhttp3.RequestBody.Companion.asRequestBody
import java.io.File

suspend fun uploadFile(
    url: String,
    token: String,
    file: File,
    mimeType: String,           // "image/jpeg" or "image/png"
    extraFields: Map<String, String> = emptyMap()
): String {
    val client = OkHttpClient()

    val bodyBuilder = MultipartBody.Builder()
        .setType(MultipartBody.FORM)
        .addFormDataPart(
            "file",                  // field name — must be "file"
            file.name,
            file.asRequestBody(mimeType.toMediaType())  // raw binary
        )

    // Add extra text fields
    extraFields.forEach { (key, value) ->
        bodyBuilder.addFormDataPart(key, value)
    }

    val request = Request.Builder()
        .url(url)
        .addHeader("Authorization", "Bearer $token")
        // Do NOT set Content-Type — OkHttp sets it with boundary
        .post(bodyBuilder.build())
        .build()

    val response = client.newCall(request).execute()
    if (!response.isSuccessful) {
        throw Exception("Upload failed: ${response.code} ${response.body?.string()}")
    }
    return response.body?.string() ?: ""
}

// ─── Usage: Upload child profile image ─────────────────
val imageFile = File(imageUri.path!!)  // convert content:// URI first
val result = uploadFile(
    url = "$baseUrl/api/v1/children/$childId/profile-image",
    token = jwtToken,
    file = imageFile,
    mimeType = "image/jpeg"
)

// ─── Usage: Upload gallery photo ────────────────────────
val result = uploadFile(
    url = "$baseUrl/api/v1/photos/upload",
    token = jwtToken,
    file = imageFile,
    mimeType = "image/jpeg",
    extraFields = mapOf(
        "class_ids" to """["$classId1","$classId2"]"""  // JSON string
    )
)

⚠️ Android: Converting content:// URI to File

Android image pickers return content:// URIs, not file paths. You must copy to a temp file first:

fun uriToFile(context: Context, uri: Uri): File {
    val inputStream = context.contentResolver.openInputStream(uri)!!
    val tempFile = File.createTempFile("upload_", ".jpg", context.cacheDir)
    tempFile.outputStream().use { output ->
        inputStream.copyTo(output)
    }
    inputStream.close()
    return tempFile
}

12. Common Pitfalls & Troubleshooting

Pitfall 1: Sending Base64 Instead of Binary

Symptom: 422 Unprocessable Entity — "field required" for file

Cause: Sending {"file": "data:image/jpeg;base64,..."} as JSON body

Fix: Use multipart/form-data with raw binary file data. The server uses FastAPI's UploadFile = File(...) which only accepts multipart uploads.

Pitfall 2: Manually Setting Content-Type Header

Symptom: 422 error even though you're using FormData

Cause: Setting Content-Type: multipart/form-data manually without the boundary

Fix: Remove the Content-Type header entirely. Let your HTTP client generate it with the correct boundary string.

Exception: In Swift/Kotlin where you build the multipart body manually, you MUST set the Content-Type with the boundary you used: multipart/form-data; boundary=YourBoundary

Pitfall 3: Wrong Form Field Name

Symptom: 422 — [{"loc": ["body", "file"], "msg": "field required"}]

Cause: Using field name "image", "photo", or "attachment" instead of "file"

Fix: All three upload endpoints expect the field name to be exactly "file":

formData.append('file', ...)   // ✅ correct
formData.append('image', ...)  // ❌ wrong
formData.append('photo', ...)  // ❌ wrong

Pitfall 4: Android content:// URI

Symptom: File not found or empty upload on Android

Cause: Android image pickers return content:// URIs which can't be read directly as file paths

Fix: Copy the content to a temporary file first (see Kotlin section above), then upload the temp file.

Pitfall 5: iOS file:// Prefix

Symptom: Upload works on Android but fails on iOS

Cause: Some iOS image pickers include the file:// prefix, others don't

Fix: Normalize the URI before uploading:

const normalizedUri = Platform.OS === 'ios'
  ? uri.replace('file://', '')
  : uri;

Pitfall 6: Camera Photos Exceeding 10 MB

Symptom: 422 — "File too large (max 10MB)"

Cause: Modern phone cameras produce 5-15 MB JPEG files

Fix: Compress before uploading:

  • React Native (expo-image-picker): quality: 0.8
  • Flutter (image_picker): imageQuality: 80, maxWidth: 2000
  • Swift: image.jpegData(compressionQuality: 0.8)
  • Kotlin: Bitmap.compress(JPEG, 80, outputStream)

Pitfall 7: Image Retrieval Returns 404

Symptom: Image URL from response returns 404

Cause: The response contains a relative URL like /api/v1/photos/xyz/image

Fix: Always prepend your base URL:

const fullUrl = `${BASE_URL}${photo.image_url}`;
// e.g. "https://your-domain.com/api/v1/photos/xyz/image"
