1. What Is "Text Class"?

"Text Class" is a feature on the Coach Dashboard's Check-In page that allows coaches to send a bulk SMS text message to all parents of students enrolled in the currently selected class.

Purpose & Use Cases

  • Class announcements: "Practice cancelled today due to weather"
  • Schedule changes: "Tomorrow's practice moved to Field B at 4:30 PM"
  • Reminders: "Don't forget to bring water bottles for Saturday's game"
  • Check-in reminders: "Please check in your child when you arrive"
  • Urgent notifications: "Practice ending early today — pickup at 5 PM"

Who Can Use It

Coach Admin Owner

Parents cannot use this feature. It is exclusively for staff members.

3. How It Works End-to-End (Backend)

Complete Flow Diagram

Coach clicks "Text Class" button
→ Modal opens (NOT YET BUILT) with message input textarea
→ Coach types message (max 160 characters = 1 SMS)
→ Coach clicks "Send"
→ Frontend sends POST /api/v1/check-in/text-class
    with { class_id, message }

↓ Backend Processing:
1. Verify user is Coach/Admin/Owner
2. Verify class exists
3. Load all active enrollments for that class
4. For each enrollment → load child → load parent (user)
5. Collect all parent phone numbers

↓ SMS Sending:
6. Format message: "CSF - {Class Name}\nCoach {Name}: {message}"
7. Call Twilio API for each phone number individually
8. Track sent/failed counts

↓ Response to Frontend:
{ sent_count: 15, failed_count: 2, total: 17, message: "..." }

SMS Message Format

The backend formats every SMS with a standard header:

CSF - Soccer Stars (Mon/Wed 4-5 PM)
Coach John Smith: Practice cancelled tomorrow due to rain. See you Wednesday!

This format is hardcoded in SMSService.send_class_announcement() — the coach only types the message body, and the server prepends the class name and coach name automatically.

Character Limit

The message field has a max length of 160 characters (validated by the schema). This is the standard single SMS segment length. Note that the formatted message (with header) may exceed 160 chars, in which case Twilio automatically splits it into multi-segment SMS and the send_sms() method truncates to 160 chars total. This means the coach's actual message may be truncated if the header is long.

4. API Endpoint Reference

POST /api/v1/check-in/text-class

Send an SMS text message to all parents of students enrolled in a specific class.

Who Can Use This

Coach Admin Owner

Authentication

Authorization: Bearer <jwt_token> — Required

Request Body — application/json

FieldTypeRequiredValidationDescription
class_id string (UUID) Yes Must be valid class UUID The class whose parents will receive the SMS
message string Yes 1–160 characters The message content. Server prepends class name + coach name automatically.

Example Request

{
  "class_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "message": "Practice cancelled tomorrow due to weather. Stay safe!"
}

Success Response — 200 OK

{
  "sent_count": 15,       // number of SMS successfully sent
  "failed_count": 2,      // number of SMS that failed (bad phone numbers, etc.)
  "total": 17,             // total parents with phone numbers
  "message": "Message sent to 15 out of 17 parents"
}

Response Field Details

FieldTypeDescription
sent_countintNumber of SMS messages successfully delivered to Twilio
failed_countintNumber of SMS that failed (invalid phone, Twilio error, etc.)
totalintTotal number of unique parent phone numbers found for the class
messagestringHuman-readable summary message for display in UI

Special Case: No Phone Numbers

If no parents in the class have phone numbers on file, the response will be:

{
  "sent_count": 0,
  "failed_count": 0,
  "total": 0,
  "message": "No parent phone numbers available"
}

5. SMS Service (Twilio) Details

Architecture

File: csf_backend/app/services/sms_service.py

The SMS service is a singleton class (SMSService) that wraps the Twilio REST API.

PropertyValue
SMS ProviderTwilio (twilio.rest.Client)
InitializationSingleton via get_sms_service()
Graceful DegradationIf Twilio credentials are missing, self.enabled = False and all send methods raise ValueError
Phone FormatE.164 required (e.g., +12125551234). Auto-prepends +1 (US) if missing.
Max Message Length160 characters per SMS segment (truncated with "..." if exceeded)
Sending MethodSequential — one API call per phone number (no batch/queue)

How send_class_announcement() Works

def send_class_announcement(self, class_name, coach_name, phone_numbers, message):
    # Step 1: Format message with header
    formatted_message = (
        f"CSF - {class_name}\n"
        f"Coach {coach_name}: {message}"
    )

    # Step 2: Send to all parents (one by one)
    return self.send_bulk_sms(phone_numbers, formatted_message)

Important: Message Truncation Issue

The coach's message is limited to 160 chars, but the formatted message (with "CSF - {class}\nCoach {name}: " prefix) can easily exceed 160 chars total. The send_sms() method truncates the entire formatted message to 160 chars, which means part of the coach's actual message may be cut off.

Example: If class name is "Soccer Stars (Mon/Wed 4-5 PM)" (30 chars) and coach name is "John Smith" (10 chars), the header alone is ~50 chars, leaving only ~110 chars for the coach's message before truncation.

Other SMS Methods Available in the Service

The SMS service has several other methods that can be called from other parts of the system:

MethodPurposeCurrently Used?
send_sms()Send single SMS to one numberInternal (called by others)
send_bulk_sms()Send same message to multiple numbersInternal (called by class announcement)
send_class_announcement()Bulk SMS with class/coach headerYes — by text-class endpoint
send_check_in_reminder()Remind parent to check in childNot called from any endpoint
send_attendance_alert()Notify parent of child's absenceNot called from any endpoint
send_badge_earned_notification()Notify parent of badge earnedNot called from any endpoint
send_event_reminder()Bulk event remindersNot called from any endpoint

Only send_class_announcement() is actively used. The other methods are available but not wired to any API endpoint yet.

8. Mobile App Implementation Guide

8.1 API Call — React Native Example

/**
 * Send SMS to all parents in a class.
 * This is a JSON POST (NOT multipart/form-data).
 */
async function textClass(classId, message, token) {
  const response = await fetch(`${BASE_URL}/api/v1/check-in/text-class`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      class_id: classId,
      message: message,     // max 160 chars
    }),
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.detail || 'Failed to send message');
  }

  return await response.json();
  // Returns: { sent_count, failed_count, total, message }
}

8.2 Flutter / Dart Example

import 'dart:convert';
import 'package:http/http.dart' as http;

Future<Map<String, dynamic>> textClass({
  required String classId,
  required String message,
  required String token,
}) async {
  final response = await http.post(
    Uri.parse('$baseUrl/api/v1/check-in/text-class'),
    headers: {
      'Authorization': 'Bearer $token',
      'Content-Type': 'application/json',
    },
    body: jsonEncode({
      'class_id': classId,
      'message': message,
    }),
  );

  if (response.statusCode != 200) {
    throw Exception('Failed: ${response.body}');
  }
  return jsonDecode(response.body);
}

9. Twilio Configuration Requirements

Required Environment Variables (Backend)

VariableDescriptionExample
TWILIO_ACCOUNT_SID Twilio Account SID ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_AUTH_TOKEN Twilio Auth Token xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_PHONE_NUMBER Your Twilio phone number (E.164) +12125551234

If any of these are empty or missing: The SMS service initializes with enabled = False and all send operations raise ValueError: SMS service is not enabled.

Currently configured: All three default to empty string ("") in core/config.py. The Twilio credentials must be set in the environment (or .env file) for SMS to work.

Parent Phone Number Requirements

  • Phone numbers are stored in the User (parent) model's phone field
  • The SMS service auto-prepends +1 (US country code) if the number doesn't start with +
  • Parents without a phone number on file are silently skipped (not counted in total)
  • The feature only texts parents of actively enrolled students (enrollment status = "active", not deleted)
